## Arquitetura

O projeto segue uma arquitetura de separação entre **UI** e **lógica**:

| Pasta | Responsabilidade |
|-------|------------------|
| `components/` | Componentes do projeto (UI + estado local mínimo) |
| `components/ui/` | Primitivos de UI (shadcn/ui) — **não modificar** |
| `hooks/` | Hooks customizados (lógica reutilizável) |
| `lib/` | Funções utilitárias e configurações |
| `types/` | Definições TypeScript globais |
| `contexts/` | Providers de estado global via React Context |

## Princípios

1. **Layouts são declarativos** — `page.tsx` e `layout.tsx` apenas recebem componentes, sem lógica pesada
2. **Lógica em hooks/lib** — Funções puras e hooks encapsulam comportamentos
3. **Types centralizados** — Interfaces compartilhadas ficam em `types/`
4. **UI primitivos intocáveis** — Nunca adicionar código em `components/ui/`
