## Architecture

The project follows an architecture separating **UI** from **logic**:

| Folder | Responsibility |
|--------|----------------|
| `components/` | Project components (UI + minimal local state) |
| `components/ui/` | UI primitives (shadcn/ui) — **do not modify** |
| `hooks/` | Custom hooks (reusable logic) |
| `lib/` | Utility functions and configurations |
| `types/` | Global TypeScript definitions |
| `contexts/` | Global state providers via React Context |

## Principles

1. **Layouts are declarative** — `page.tsx` and `layout.tsx` only receive components, no heavy logic
2. **Logic in hooks/lib** — Pure functions and hooks encapsulate behaviors
3. **Centralized types** — Shared interfaces live in `types/`
4. **Untouchable UI primitives** — Never add code to `components/ui/`
